# 천재교육 EduCrew AI (자동 출제 시스템)

## 📌 프로젝트 소개 (What is this app?)
천재교육 콘텐츠팀의 **문항 출제 → 검수 → 해설 생성 파이프라인**을 자동화한 데스크톱 AI 애플리케이션입니다. 
다양한 AI 에이전트들이 협력하여 고품질의 교육 콘텐츠를 생성하며, 수학 및 영어 과목을 완벽하게 지원하고 미래의 타 과목(과학 등) 확장까지 고려하여 설계되었습니다.

## 🎯 기획 의도 (Why was it built?)
1. **제작 비용 및 시간 혁신**: 단순 출제와 해설 작업은 무료 로컬 모델을 활용하여 토큰 비용을 완전히 제거하고, 논리적 오류를 잡아내는 섬세한 '검수' 단계에만 GPT-4o-mini를 적용하여 품질과 가성비를 동시에 잡았습니다.
2. **다과목 확장성 보장**: 단일 과목(수학) 전용으로 짜여있던 코드를 모듈화하여, 새로운 과목의 프롬프트와 페르소나만 추가하면 즉시 파이프라인이 동작하도록 **과목 플러그인(Subject Plugin)** 구조를 확립했습니다.
3. **투명한 자동화 및 최종 결과물 추출**: 에이전트들의 작업 상태를 UI로 실시간 추적할 수 있으며, 최종 산출물을 인쇄 및 배포가 가능한 전문적인 레이아웃의 PDF 시험지로 자동 변환합니다.

## 🛠 기술 스택 (Technology Stack)
- **AI / LLM Orchestration**: `CrewAI` (멀티 에이전트 기반 태스크 체이닝)
- **AI Models**:
  - `Qwen3:4b` (Ollama 로컬 서버): 출제 및 해설 작성 (비용 절감)
  - `GPT-4o-mini` (OpenAI API): 정답/논리 검수 및 최종 안전성 감수 (정밀도 향상)
- **Frontend**: `HTML5`, `CSS3`, `JavaScript`, `MathJax` (수식 렌더링)
- **Desktop GUI**: `pywebview` (Python 기반의 경량화된 웹뷰 데스크톱 앱)
- **Backend Core**: Python 3
- **PDF Generation**: `reportlab` (한글 폰트 및 과목별 커스텀 레이아웃 지원)

## 🏗 아키텍처 구조 (Architecture)
시스템은 UI와 비즈니스 로직이 완전히 분리되어 있으며, 과목 플러그인 구조를 채택하고 있습니다.

### 1. 디렉토리 구조
```text
chunjae_crewai/
├── core/                # 과목과 무관한 공통 핵심 엔진
│   ├── pipeline.py      # CrewAI 오케스트레이터
│   ├── pdf_generator.py # PDF 생성 모듈 (과목별 레이아웃 분기)
│   ├── safety_review.py # 최종 안전성 및 논리 감수
│   └── llm_registry.py  # 하이브리드 LLM 통합 관리
├── subjects/            # 과목별 독립 플러그인 (프롬프트, 페르소나, 메타데이터)
│   ├── base.py          # 과목 공통 추상 인터페이스 (SubjectBase)
│   ├── math/            # 수학 과목 플러그인
│   └── english/         # 영어 과목 플러그인
├── ui/                  # 독립적인 프론트엔드 (웹뷰 UI)
├── gui.py               # 데스크톱 앱 진입점 및 pywebview 라우터
└── main.py              # 하위 호환성 래퍼 및 코어 파이프라인 호출부
```

### 2. 하이브리드 LLM 파이프라인 흐름
1. **문제 출제 (로컬 AI)**: 출제 마스터 에이전트가 단원과 유형에 맞는 문항을 생성합니다.
2. **논리 검수 (GPT)**: 베테랑 검수 에이전트가 수학적 계산 오류나 논리적 모순, 정답 유출 여부를 엄밀히 검증합니다.
3. **해설 작성 (로컬 AI)**: 검증을 통과한 문제에 대해 친절한 해설가 에이전트가 개념과 풀이 과정을 작성합니다.
4. **최종 감수 (GPT)**: `safety_review` 엔진이 최종적으로 정답 일치 여부를 한 번 더 교차 검증합니다.
5. **결과 출력**: UI 렌더링 및 PDF 파일 변환.

## 🔐 보안 및 API 키 관리
- OpenAI API 키(`OPENAI_API_KEY`)는 코드베이스에 절대 하드코딩되지 않습니다.
- `.env` 파일을 통해 환경 변수 형태로 로드되며, 해당 파일은 `.gitignore`에 의해 버전 관리에서 안전하게 제외됩니다.
