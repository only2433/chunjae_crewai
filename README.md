# 🎓 천재교육 EduCrew AI : 차세대 멀티 에이전트 자동 출제 시스템

> **로컬 오픈소스 LLM의 '가성비'와 상용 API의 '정밀함'을 융합하여, 비용은 최소화하고 교육 콘텐츠의 품질은 극대화한 차세대 에듀테크 파이프라인**
> 
> 수학과 영어 등 교과목별 특성에 맞춘 다수의 AI 에이전트(CrewAI)가 서로 협력, 견제, 검증하며 시중 교재 수준의 고품질 문제를 끝없이 쏟아냅니다.

---

**EduCrew AI**는 천재교육 콘텐츠팀의 핵심 워크플로우(문항 출제 → 논리 검수 → 해설 생성)를 완전히 자동화하는 데스크톱 기반 AI 애플리케이션입니다. 

기존 AI 생성 서비스들이 단일 모델의 한계로 인해 '계산 오류', '성별 모호성', '정답 유출' 등의 환각(Hallucination) 현상을 겪는 반면, 본 시스템은 **'문제 출제자', '전문 감수관', '해설 강사'** 등 고유의 페르소나를 지닌 여러 AI 에이전트를 조율하는 **Multi-Agent Orchestration**을 도입하였습니다. 출제 작업은 로컬 모델에 맡겨 토큰 비용을 0원으로 만들고, 까다로운 논리 검수에만 GPT-4o-mini를 투입하는 **하이브리드 아키텍처**로 최소 비용, 최고 품질을 달성했습니다.

---

## 🌟 비즈니스 가치 (Executive Summary)

- 💰 **파격적인 비용 절감 (Zero-Cost Generation)**: 문제와 해설을 끝없이 만들어내는 '출제/해설' 작업은 데스크톱 내장 로컬 AI가 전담하여 토큰 API 비용을 **0원**으로 만들었습니다.
- 🛡️ **결점 없는 품질 보증 (Zero-Defect Quality)**: 논리적 모순, 수식 오류, 어법 오류를 잡아내는 섬세한 '검수' 단계에만 클라우드 AI를 선택적으로 투입했습니다.
- 🚀 **무한한 확장성 (Subject Plugin Architecture)**: 단일 과목에 종속되지 않는 플러그인 아키텍처를 도입하여, 향후 국어, 과학, 사회 등 어떤 과목이든 단 5분 만에 파이프라인에 추가할 수 있습니다.
- 🖨️ **즉시 배포 가능한 산출물 (Print-Ready PDF)**: 10문제가 담긴 모의고사를 출제하고, 천재교육 스타일의 세련된 2단 레이아웃 PDF로 자동 조판해 줍니다.

## ⚙️ 핵심 기술 스택

| 분류 | 기술 | 역할 |
|------|------|------|
| **언어** | Python 3.10+, JavaScript | 백엔드 파이프라인 엔진 및 프론트엔드 로직 |
| **Orchestration** | CrewAI | 멀티 에이전트 생성, 페르소나 부여 및 태스크 체이닝 |
| **Local LLM** | Ollama (`qwen3:4b`) | 비용 발생 없는 무제한 문제 출제 및 상세 해설 작성 |
| **Cloud LLM** | OpenAI API (`gpt-4o-mini`) | 고도의 정밀성이 요구되는 논리 검증 및 2중 안전성 감수 |
| **Frontend UI** | HTML5, CSS3, MathJax | pywebview 기반 데스크톱 앱 웹뷰, 수식 동적 렌더링 |
| **PDF Generation**| Reportlab | 한글 폰트 적용 및 과목별 특수 레이아웃 자동 조판 |

## 🔄 하이브리드 자동화 파이프라인

```text
Web Dashboard에서 과목 / 학년 / 단원 선택 및 '출제 시작'
    │
    ▼
[1단계] 출제 마스터 에이전트 (Local LLM: qwen3)
    - 교과과정 단원과 유형(계산형/문장제/독해형 등)에 맞춰 문제 초안 생성
    - 수식의 경우 엄격한 표준 LaTeX 포맷 준수
    │
    ▼
[2단계] 과목별 전담 검수관 에이전트 (Cloud LLM: gpt-4o-mini)
    - 수학: AI가 직접 문제를 풀어보고 1~5번 보기 안에 자신의 답이 존재하는지 수식/논리 검증 (Self-Verification)
    - 영어: 대명사 성별 모호성, 중복 정답 가능성, 교육과정 어휘 적합성 치밀하게 교정
    │
    ▼
[3단계] 핵심 어법/개념 해설가 에이전트 (Local LLM: qwen3)
    - 통과된 문제에 대해 핵심 개념 → 단계별 풀이 → 최종 정답의 구조로 맞춤형 해설 작성
    - 영어의 경우 오답 보기가 왜 틀렸는지까지 1줄씩 상세 서술
    │
    ▼
[4단계] 최종 안전성 감수 (Cloud LLM: logic_safety_review)
    - 정답 노출 여부 및 최종 해설 일치 여부를 파이썬 코드 레벨에서 2중 차단 및 정규식 정제
    │
    ▼
[결과 렌더링] UI 화면 실시간 렌더링 및 PDF (Reportlab) 익스포트
```

## 🏗 목표 디렉토리 구조 (과목 플러그인 아키텍처)

초기 모놀리식 구조에서 탈피하여, 과목별로 비즈니스 로직을 철저히 분리한 **모듈형 플러그인 아키텍처**를 완성했습니다.

```text
chunjae_crewai/
├── core/                          # 🧠 과목 무관 공통 엔진
│   ├── pipeline.py                # Crew 실행 오케스트레이터 (공통)
│   ├── pdf_generator.py           # PDF 생성 (수학/영어 레이아웃 동적 분기)
│   ├── safety_review.py           # logic_safety_review (공통 2중 검수)
│   └── llm_registry.py            # 하이브리드 LLM 인스턴스 중앙 관리
│
├── subjects/                      # 🧩 과목별 플러그인 디렉토리
│   ├── base.py                    # SubjectBase 추상 클래스 (플러그인 표준 인터페이스)
│   │
│   ├── math/                      # 📐 수학 플러그인
│   │   ├── config.json            # 수학 메타데이터 (학년, 단원, 문제 유형)
│   │   ├── agents.py              # 수학 특화 에이전트 페르소나
│   │   └── tasks.py               # 수학 특화 태스크 프롬프트
│   │
│   └── english/                   # 🔤 영어 플러그인
│       ├── config.json            # 영어 메타데이터 (유형, 학년, 수준)
│       ├── agents.py              # 영어 원어민 특화 에이전트 페르소나
│       └── tasks.py               # 영어 독해/어법/문법 프롬프트
│
├── ui/                            # 🎨 완전히 분리된 프론트엔드
│   └── index.html                 # 과목 탭 UI 및 CSS 스타일링
├── gui.py                         # 🚀 데스크톱 앱 실행 진입점 및 pywebview 라우터
└── main.py                        # 파이프라인 진입점
```

## 🎯 대상 사용자
- **콘텐츠 출제 연구원** — 기존 대비 10배 이상의 속도로 문항 초안을 확보하고 편집 작업에만 집중하고 싶은 연구원.
- **수학/영어 학원 강사** — 학생 수준에 맞는 커스텀 모의고사 시험지(PDF)를 원클릭으로 생성하여 즉시 배포해야 하는 교육 현장.
- **에듀테크 서비스 기획자** — 로컬 AI와 상용 AI의 비용 효율적인 하이브리드 파이프라인 아키텍처를 자사 서비스에 이식하고자 하는 리더.
